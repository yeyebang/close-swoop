# 重构开发计划

## 目标

将"尾盘涨停扫描器"从基于日线+规则打分，重构为基于**分钟级数据 + LightGBM 回归 + 滚动窗口回测**的量化策略系统。

核心转变：
- 从"预测涨停"（<0.5% 基准率，不可预测）转向"预测次日开盘收益"（~55% 正收益概率，可学习）
- 从日线代理回测转向真实的 walk-forward 回测
- 从规则打分转向模型排序

## 数据源

- AKShare 1min/5min K 线：`stock_zh_a_hist_min_em` / `stock_zh_a_hist_5min_em`
- AKShare 日K 保留（中长期趋势）
- AKShare 板块/概念数据
- 大盘情绪：涨跌家数、涨停家数、成交额

## 关键设计决策

### 1. 为什么用 LightGBM 而不是深度学习

- 数据量：每天 ~2000 只股票 × N 天 = 几万条样本，深度学习需要十万级
- 特征类型：表格数据，LightGBM 天然适合
- 可解释性：需要知道哪些特征在起作用
- 训练速度：分钟级 vs 小时级

### 2. 为什么预测次日收益而不是涨停

| 指标 | 预测涨停 | 预测次日收益 |
|------|----------|-------------|
| 基准率 | <0.5% | ~55% 正收益 |
| 信号信噪比 | 极低 | 中等 |
| 可学习性 | 几乎不可学 | 可学 |
| 实用价值 | 低（太少了） | 高（每天选 5-10 只） |

### 3. 分钟数据粒度选择

- **1min**：噪音大，但信息最全
- **5min**：噪音小，特征更稳定
- **方案**：主用 5min，关键信号用 1min 验证

### 4. Walk-forward 回测框架

```
T=1:   用 [1..60] 天训练 → 预测 T+1
T=61:  用 [2..61] 天训练 → 预测 T+1
T=62:  用 [3..62] 天训练 → 预测 T+1
...
```

- 每次只用训练时的数据，不 lookahead
- 滑动窗口大小可配（默认 60 天）
- 验证集：最近 10 天

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| AKShare 分钟接口不稳定 | 数据缺失 | 多源备用 + 缓存降级 |
| 模型过拟合 | 实盘效果差 | 严格 walk-forward + 特征重要性监控 |
| 数据量不足 | 模型效果差 | 先跑 30 天再训练，逐步积累 |
| 接口限流 | 拉取慢 | 增量更新 + 缓存 |

## 验收标准

1. 回测显示：Top 5 候选，次日正收益概率 > 55%
2. 回测显示：平均次日收益 > 0
3. 扫描流程 < 5 分钟（含数据获取）
4. 模型每天自动更新
5. 新旧模式可切换，不影响现有功能
